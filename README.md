# sky-prd-ui-prototype-skill

基于 PRD 文档自动生成高保真 HTML+CSS+JS 效果图/交互原型的 Codex Skill。

本仓库目前以 `SKILL.md` 形式定义了一套“从 PRD → 高保真可预览原型”的完整规范，后续可以按此规范逐步补充脚本实现。

## 功能定位

- 输入：一份结构化较好的 PRD 文档（可以是由 `sky-demand-prd-builder-skill` 生成，也可以是任意标准化 PRD），外加少量参数。
- 输出：一组高保真 HTML+CSS+JS 页面，用于在浏览器中直接预览产品界面和主流程。
- 核心目标：
  - 缩短“从 PRD 到可见界面”的距离；
  - 帮助产品/设计/研发在评审阶段快速对齐对“长什么样”的预期；
  - 为后续专业视觉设计和前端实现提供参考，而不是替代它们。

## 关键特性（来自 SKILL 规范）

详细规则见根目录的 `SKILL.md`，这里只总结核心要点：

- **平台必问**：生成前必须询问目标平台，只生成一个端（PC 客户端 / Web / 移动端 / 电视大屏），除非 PRD/用户明确要求多端或响应式。
- **仅读 PRD，不改 PRD**：从 PRD 中解析页面、模块和流程，不回写或修改 PRD 内容。
- **页面自动识别**：根据 PRD 的“页面/模块/流程/信息架构”等章节，先生成“页面与路由清单草案”，让用户有机会纠偏，再渲染 HTML。
- **页面类型→布局模板映射**：内置列表页、表单/配置页、详情/预览页、仪表盘、弹框、空状态等页面类型，并根据平台做适配。
- **统一视觉与组件体系**：
  - 一个 PRD 对应一套主题（颜色、字体、按钮、卡片等）；
  - 用户不指定时，自动选择一套“默认最美观”的中性主题；
  - 所有页面共用统一的 `style.css` 和组件样式。
- **HTML+CSS+JS 输出**：
  - 目录约定：`prdoutput/ui/`，包含 `index.html`、各页面 HTML、`style.css`、`script.js`；
  - JS 只做轻交互：导航高亮、显示/隐藏、步骤切换、loading 等，不实现真实业务逻辑。
- **交互与状态**：为关键控件提供 hover/active/disabled/loading 等基础状态样式，支持主流程的页面跳转链路。
- **可访问性与隐私边界**：
  - 尽量遵守基础无障碍实践（label、aria-label、颜色对比度等），不过度追求标准全覆盖；
  - 示例数据一律为虚构/脱敏，不从环境读取真实用户数据。
- **审查与纠偏机制**：
  - 生成后自动做一次“HTML ↔ PRD 一致性自审查”，检查页面覆盖、主流程可达、关键元素是否体现等；
  - 输出结果摘要，标出遗漏/多余/存疑部分，提示产品/设计进行纠偏；
  - 支持多轮迭代：优先只改指定页面/区域，保持整体主题和结构稳定。
- **示例 PRD 映射**：
  - `SKILL.md` 使用“视频字幕截帧拼图”作为示例，演示如何从 PRD 推导出“选择视频 → 配置与进度 → 结果预览”等页面；
  - 示例仅用于说明方法论，本 skill 对任何结构化 PRD 都适用。

## 目录结构

当前仓库结构非常简单：

- `SKILL.md`：本 skill 的完整规范文档，定义输入输出、约束、边界和示例。
- `README.md`：当前说明文档。

后续可以按需增加：

- `scripts/`：实现从 PRD 文本到 HTML+CSS+JS 文件的自动生成脚本。
- `templates/`：通用页面模板、组件片段等。
- `examples/`：示例 PRD 与对应生成结果，便于回归和演示。

## 使用方式（在 Codex / AI_Skills 中）

1. 将本仓库安装或拷贝到 Codex 的 skills 目录下，保持 `SKILL.md` 在 skill 根目录。
2. 在对话中显式提到本 skill 名称（例如：`sky-prd-ui-prototype-skill`），并提供：
   - 目标平台（PC/Web/移动端/TV）；
   - PRD 文本（可以是前一个 PRD skill 的输出，也可以是任意标准化 PRD）；
   - 可选的设计风格/主题色要求。
3. 按 `SKILL.md` 中的流程，模型会先给出“页面清单草案”和纠偏入口，确认后再生成 HTML+CSS+JS 原型，并执行一致性审查。
4. 如需进一步开发，可在本仓库新增脚本实现具体生成逻辑，保持与 `SKILL.md` 的行为规范一致。
